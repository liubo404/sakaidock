Your localkernel.properties file should contain the following key/value pair: component.locations=maven-repo:org.sakaiproject.kernel2.agnostic:kernel:0.1-SNAPSHOT:jar;maven-repo:org.sakaiproject.kernel2.agnostic:component-sample:0.1-SNAPSHOT:jar; *Step 3: Run the Bootstrap* * Note: make sure you unset any CATALINA environment variables you may have such as $CATALINA_HOME or $CATALINA_LOGDIR. mvn -Prun clean install The Maven "run" profile will deploy and start Tomcat with K2 up and running along with a sample webapp ("webapp-sample"): http://localhost:8080/webapp-sample/helloworld.jsp http://localhost:8080/webapp-sample/status.jsp The running Tomcat instance is located in the /bootstrap folder: trunk/bootstrap/target/runtime In the bootstrap folder you will also find a localkernel.properties file that contains a local override for the components that are started. By default it lists the following component locations built in step 2: component.locations=maven-repo:org.sakaiproject.kernel2.agnostic:kernel:0.1-SNAPSHOT:jar; Each component is specified as follows: 1. maven-repo is the protocol being used in the URL in this case into the maven repo, but could have been a filesystem path. 2. org.sakaiproject.kernel2.agnostic is the Group ID of the component, in this case the Kernel Component 3. kernel is the Artifact ID 4. 0.1-SNAPSHOT is the Version 5. jar is the packaging which should be a jar file More than one component can be listed using a semi-colon ";" as a separator (all on one line or use the \ mechanism in properties to spread over multiple lines.).
